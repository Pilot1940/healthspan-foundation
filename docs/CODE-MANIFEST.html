Migrations (PostgreSQL DDL)

SQL migrations/029_telegram_ingestion.sql

Phase 1 Telegram schema — five new tables, all RLS-protected.

  • telegram_link_codes — one-time activation codes (maintainer mints, recipient sends to bot). Credential: never log in plaintext.
  • telegram_identities — chat_id → profile_id binding. Status: pending → active → revoked. RLS: each profile sees own row; maintainer sees all.
  • telegram_processed_updates — inbound idempotency latch. Written ONLY after media_inbox insert succeeds (Telegram retry safety).
  • media_inbox — durable inbound queue drained by Phase-3 Routine. caption stored as DATA only (injection rule).
  • push_log — outbound push idempotency + debounce ledger. Debounce key: (profile_id, push_type, subject_id).
SQL migrations/030_push_enhancements.sql Phase 2

Adds push_log.dedup_value and seeds all push thresholds into system_config.

  • push_log.dedup_value numeric — metric at push time; enables material-change re-score gate. NULL for non-numeric push types.
  • 9 push.* system_config rows: debounce window, quiet hours, recovery critical threshold, HRV crash %, material change %, dead-man hours, reconcile default + max widen hours.
  • Verify: confirms column present + all 9 config rows active.

Edge Functions (Deno/TypeScript)

TS Edge Fn supabase/functions/whoop-webhook/index.ts Phase 2 updated

Receives WHOOP webhook events; verifies HMAC-SHA256 signature; upserts to whoop_workouts / whoop_cycles / whoop_sleeps. Phase 2: composes + sends Telegram push after each upsert. Dead-man check on every event.

  • verify() — constant-time HMAC-SHA256 signature check.
  • mapWorkout / mapSleep / mapCycle — WHOOP API → DB row mappers.
  • refreshPriorCycle() — re-fetches the just-closed cycle's final strain (WHOOP emits no cycle.updated webhook).
  • telegramSend() — best-effort sendMessage; failures never kill the upsert.
  • loadPushConfig() — reads all push.* rows from system_config.
  • export localHour(timezone, now) — UTC→local hour; accepts IANA names + UTC offset strings. Tested.
  • export deadManThreshold(lastDataMs, thresholdHours, nowMs) — pure threshold check. Tested.
  • export decidePush(opts) — returns "send" | "suppress_debounce" | "suppress_quiet" | "suppress_unchanged". Priority: critical → debounce+material-change → quiet hours → send. Tested.
  • export composePush(pushType, row, isMinor) — short templated message; minor framing (no deficit language). Tested.
  • isHrvCrash() — queries last 7 cycles for HRV baseline; fires critical if drop ≥ push.hrv_crash_pct.
  • checkDeadMan() — checks all active profiles; alerts PC if any profile exceeds push.dead_man_hours since last records_upserted > 0. Suppressed for 22h per profile.
  • Push block runs in detached async closure — never delays the 200 response.
TS Edge Fn supabase/functions/telegram-webhook/index.ts

Inbound Telegram message handler. Auth, dedup, media→Storage, enqueue.

  • verifySecretToken() — constant-time X-Telegram-Bot-Api-Secret-Token check.
  • guessKind(caption) — keyword classifier for food/workout/lab/dexa/unknown. Caption is DATA only — never executed.
  • storeMedia() — Telegram getFile → download → upload to health-media private bucket.
  • Link-code activation: /start <code> or bare code validates against telegram_link_codes.
  • Identity lookup: chat_id → telegram_identities → profile_id. Unknown → onboarding + maintainer alert. Revoked → reject.
  • Enqueue: media_inbox INSERT first; telegram_processed_updates INSERT only after (idempotency ordering).

Push Decision Flow

// whoop-webhook: after successful upsert WHOOP event arrives └─ verify HMAC signature └─ map whoop_user_id → profile_id └─ fetch + map + upsert (whoop_workouts / whoop_cycles / whoop_sleeps) └─ wearable_sync_log → status: success └─ detached async push block (never delays 200 response) │ ├─ loadPushConfig(system_config) ← all thresholds from DB ├─ lookup telegram_identities(profile_id) → chat_id, is_minor │ ├─ determine push_type + dedupValue + isCritical: │ recovery/cycle → push_type = "recovery_landed" │ isCritical = score < 34 OR HRV crash ≥ 20% of baseline │ workout → push_type = "workout_logged" │ ├─ lookup push_log(profile, type, subject_id) ORDER BY sent_at DESC LIMIT 1 │ ├─ decidePush(now, lastPush, isCritical, currentValue, timezone, config) │ ┌ CRITICAL? → "send" (always — bypasses all gates) │ ├ within debounce window? │ │ └ material change ≥ 5ppt? → "send" │ │ └ else → "suppress_debounce" │ ├ quiet hours (local time)? → "suppress_quiet" │ └ else → "send" │ ├─ if "send" → composePush(type, row, is_minor) → telegramSend(chat_id, text) │ ├─ push_log INSERT (status: sent | suppressed, dedup_value: score) │ └─ checkDeadMan: scan ALL active profiles └─ last wearable_sync_log with records_upserted > 0 > 36h ago? └─ already alerted within 22h? → skip └─ else → telegramSend(maintainer_chat, "⚠️ Dead-man: ...") + push_log INSERT

Push Thresholds (system_config — migration 030)

KeyValueMeaning
push.debounce_window_min30Minutes. Burst of same (profile, type, subject) collapses to one push, unless score changes materially.
push.quiet_hours_start22Local hour. Non-critical pushes suppressed from 10pm (evaluated in WHOOP timezone).
push.quiet_hours_end7Local hour. Non-critical pushes resume at 7am.
push.recovery_critical_threshold34Recovery % below which push is CRITICAL — sent regardless of quiet hours or debounce.
push.hrv_crash_pct20HRV drop (%) vs 7-cycle baseline that triggers CRITICAL alert.
push.material_change_pct5Score change (ppt) threshold for a re-score within debounce window to re-push.
push.dead_man_hours36Hours of no WHOOP data before dead-man alert to PC. Anchor: records_upserted > 0.
push.reconcile_default_hours48Default reconcile window. Widens per profile if last sync is older.
push.reconcile_max_widen_hours168Max reconcile window cap (7 days). Prevents unbounded backfill on dormant profiles.

⚠  Dead-man fully-silent backstop

Dead-man check runs on every WHOOP event (covers all profiles). If ALL profiles simultaneously go silent (no webhooks at all), a pg_cron job calling the edge fn endpoint is needed as a time-triggered backstop. Not yet deployed — schedule when an availability incident makes it warranted.

⚠  Altitude early-warning
