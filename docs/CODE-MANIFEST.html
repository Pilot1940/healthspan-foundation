SQL migrations/029_telegram_ingestion.sql

Phase 1 Telegram schema — five new tables, all RLS-protected.

  • telegram_link_codes — one-time activation codes (maintainer mints, recipient sends to bot).
  • telegram_identities — chat_id → profile_id binding. Status: pending → active → revoked.
  • telegram_processed_updates — inbound idempotency latch. Written ONLY after media_inbox insert succeeds.
  • media_inbox — durable inbound queue. caption stored as DATA only (injection rule).
  • push_log — outbound push idempotency + debounce ledger.
SQL migrations/030_push_enhancements.sql

Adds push_log.dedup_value and seeds all push thresholds into system_config. 9 push.* rows: debounce window, quiet hours, critical threshold, HRV crash %, material change %, dead-man hours, reconcile defaults.

SQL migrations/031_maintainer_ingest.sql

Adds media_inbox.media_group_id for album clustering, 4 system_config rows, and 3 SECURITY DEFINER RPCs for maintainer cross-profile writes from the cloud Routine.

  • media_inbox.media_group_id text NULL — Telegram album burst ID. Rows sharing this value drained as one cluster → one extraction → one log entry.
  • 4 system_config seeds: push.inbox_settle_sec=90, ingest.confidence_threshold=0.7, routine.last_fire_at, routine.fire_dedup_sec=300.
  • maintainer_ingest_food — SECURITY DEFINER. Guard: is_maintainer() AND has_profile_access(p_profile_id). High-conf → food_logs; low → stg_food_log_review.
  • maintainer_ingest_biomarker — Upserts on (profile_id, metric_definition_id, measured_at); stages on low confidence.
  • maintainer_ingest_supplement — Upserts on (profile_id, supplement_id, taken_on, source); taken_on is GENERATED, never written directly.

Monitor (Python)

PY monitor/inbox_drain.py Phase 3B complete

Fully autonomous cloud Routine entrypoint. Run with python -m monitor.inbox_drain --once — fetches settled items, clusters, extracts via Anthropic vision API, writes via SECURITY DEFINER RPCs, sends Telegram confirmations, prints JSON summary. Uses lib.db_rest (httpx only — no psycopg2).

  • run_once(db, cfg, api_key, token) → dict — full drain loop. Returns {fetched, clustered, written, staged, failed, errors:[]}.
  • vision_extract(api_key, model, image_blocks, caption, kind, now_iso) — POSTs to https://api.anthropic.com/v1/messages. Model from drain.vision_model system_config. Returns parsed JSON or {confidence:0, _error:"..."} on failure — always routes to staging rather than crashing.
  • content_cluster_ungrouped(api_key, model, items) — text-only API call groups ungrouped items by caption/kind. Single item skips API call; falls back to singletons on any error.
  • build_clusters(items) — album clusters (shared media_group_id ≥ 2) + ungrouped.
  • claim(db, item_id) — atomic PATCH pending → processing; returns False if another fire won the race.
  • get_signed_url / _download_image / _image_block — Storage access pipeline; storage_path=None skips gracefully.
  • write_food / write_biomarker / write_supplement — call SECURITY DEFINER RPC; mark rows done/staged.
  • compose_confirmation(kind, rpc_status, extracted, is_minor) — minor-safe: growth/performance framing only when is_minor=True.
  • telegram_send(token, chat_id, text) — best-effort, never raises.
  • CLI: python -m monitor.inbox_drain --once — print JSON summary, exit 0/1. Without --once: dry-run cluster summary only.

Push Decision Flow (whoop-webhook)

// whoop-webhook: after successful upsert WHOOP event arrives └─ verify HMAC signature └─ map whoop_user_id → profile_id └─ fetch + map + upsert (whoop_workouts / whoop_cycles / whoop_sleeps) └─ wearable_sync_log → status: success └─ detached async push block (EdgeRuntime.waitUntil — never delays 200) │ ├─ loadPushConfig(system_config) ← all thresholds from DB ├─ lookup telegram_identities(profile_id) → chat_id, is_minor │ ├─ determine push_type + dedupValue + isCritical │ ├─ lookup push_log(profile, type, subject_id) ORDER BY sent_at DESC LIMIT 1 │ ├─ decidePush(now, lastPush, isCritical, currentValue, timezone, config) │ ┌ CRITICAL? → "send" (always — bypasses all gates) │ ├ within debounce window? │ │ └ material change ≥ 5ppt? → "send" │ │ └ else → "suppress_debounce" │ ├ quiet hours (local time)? → "suppress_quiet" │ └ else → "send" │ ├─ if "send" → composePush(type, row, is_minor) → telegramSend(chat_id, text) │ ├─ push_log INSERT (status: sent | suppressed, dedup_value: score) │ └─ checkDeadMan: scan ALL active profiles └─ last wearable_sync_log with records_upserted > 0 > 36h ago? └─ already alerted within 22h? → skip └─ else → telegramSend(maintainer_chat, "⚠️ Dead-man: ...") + push_log INSERT
